# ChemLab (Unity 6) — план и базовая архитектура

## 1) Пошаговый план разработки

1. **Вертикальный срез (MVP, 1-2 недели)**
   - FPS-контроллер + взаимодействие ПКМ/ЛКМ.
   - 10-15 веществ (SO), 8-10 реакций (SO).
   - Контейнер (пробирка) с температурой, смешиванием и простыми VFX.
   - Инвентарь: хотбар 1-6 + Tab-инвентарь.
2. **Системная глубина (2-4 недели)**
   - Катализаторы, давление, электролиз.
   - Система открытий и журнал реакций.
   - Оборудование: горелка, холодильник, центрифуга, фильтр.
3. **Контент и баланс (2-6 недель)**
   - 60+ веществ, 100+ реакций.
   - Квесты (создай вещество X), ошибки игрока (взрывы/токсичность).
   - Скрытые реакции (пасхалки).
4. **Полировка**
   - VFX/SFX, оптимизация, UX, сохранения.

## 2) Архитектура проекта (папки)

- `Assets/ChemLab/Scripts/Core` — взаимодействие игрока, общие интерфейсы.
- `Assets/ChemLab/Scripts/Chemistry` — runtime-химия (контейнер, движок реакций).
- `Assets/ChemLab/Scripts/Chemistry/Data` — ScriptableObject данные элементов/реакций.
- `Assets/ChemLab/Scripts/Inventory` — хотбар/инвентарь/мировые предметы.
- `Assets/ChemLab/Scripts/Equipment` — лабораторное оборудование.
- `Assets/ChemLab/Scripts/UI` — UI кодекса, инвентаря, подсказок.
- `Assets/ChemLab/Scripts/Gameplay` — открытия, квесты, режимы.

## 3) SOLID-подход

- **S**: `ReactionEngine` только проверяет/применяет реакции.
- **O**: новые реакции добавляются через SO, код реактора не меняется.
- **L**: любое оборудование наследует `LabEquipmentBase`.
- **I**: UI, химия и инвентарь разделены.
- **D**: зависимости через сериализуемые ссылки и данные SO.

## 4) Что уже покрыто кодом

- Таблица веществ через `ChemicalElementData`.
- Реакции через `ChemicalReactionData` + условия (T, давление, катализатор, электролиз).
- Контейнер смеси (`ChemicalContainer`) с визуальными эффектами.
- Инвентарь (1-6 + расширенный) и swap drag-n-drop логикой.
- Оборудование: нагрев/охлаждение/центрифуга/фильтр/электролизер.
- Codex UI по Tab (элементы, реакции, подсказки).
- Система открытий.

## 5) Оптимизация

- Использовать Object Pool для VFX (дым, пузырьки, осадок).
- Реакции считать **по событию**, а не каждый кадр.
- Кэшировать lookup (словарь `elementId -> количество`) при росте контента.
- Разделить химическую симуляцию и визуализацию (меньше аллокаций GC).
- Для мультиплеера: сервер-авторитет на реакциях.

## 6) Идеи расширения

- pH-буферные системы и индикаторы.
- Радиоактивный распад во времени + защита игрока.
- Автоматические установки (pipes + логические контроллеры).
- Режим карьеры/квестов и sandbox.
- Кооперативный мультиплеер на Netcode for GameObjects.
